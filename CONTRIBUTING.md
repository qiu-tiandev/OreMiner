## Contributing to OreMiner
Thanks for your interest in contributing! Keep reading to see how you can contribute.

### Reporting Bugs
If you find a bug, please submit a report under `issues`
The report should ideally include:
* Overview of bugs
* Steps to reproduce it

### Suggesting improvements
Make OreMiner better by submitting an issue with tag `suggestions`.
Include:
* What could be implemented/optimised
* Explain the use case (if applicable)
* Examples of how the feature would work (if applicable)

### Submitting PRs
1. Look for the `todo` tag under issues for features you can work on.
2. Fork the repo and create a new branch using `git checkout -b feature or bug/your-feature-name`
3. Edit the code.

**Before submitting a PR**:
* Make Sure the changes you make works and does not break other features
**When creating a PR:**
* Reference related issues
* Explain purpose of the change
* **ONE ISSUE ONE PR**
